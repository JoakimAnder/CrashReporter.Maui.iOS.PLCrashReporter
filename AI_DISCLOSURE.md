# AI Disclosure

This project was primarily developed with AI assistance (Claude Code / Claude Sonnet). This file exists to be transparent about what that means for anyone using or contributing to it.

## What was AI-generated

- **Bindings** (`ApiDefinition.cs`, `StructsAndEnums.cs`) — fully AI-generated based on the PLCrashReporter headers.
- **CI/CD workflows** (`CI.yaml`, `CD.yaml`) — AI-assisted; structure and steps were generated and then reviewed.
- **This file** (`AI_DISCLOSURE.md`) — Generated by AI, everything except this line funilly enough.

## What was not AI-generated

- **Frameworks** (`Source/Frameworks/`) — copied directly from the [PLCrashReporter](https://github.com/microsoft/plcrashreporter) repository, not AI-generated.

## Human oversight

All AI-generated output was reviewed before being committed. That said, the bindings in particular are non-trivial to fully verify without running against a real crash report — see the known limitations below.

## Known limitations worth extra scrutiny

- The bindings cover the public API surface but may have gaps or incorrect mappings that only surface at runtime. If you hit something that looks off, please open an issue.

## Why disclose this?

Because it's the honest thing to do. AI-generated code can be perfectly fine, but you deserve to know so you can apply appropriate scrutiny — especially for a library that deals with crash data.
